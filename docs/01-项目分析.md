# VisionPass 项目分析文档

## 一、项目概述

VisionPass 是一个基于**正点原子 I.MX6U 开发板**（ARM Cortex-A7）的智能门禁系统，采用 **C/S（客户端-服务端）架构**，实现了人脸识别、RFID 刷卡、密码验证、远程控制等多种开锁方式。

### 1.1 项目定位

| 项目 | 说明 |
|------|------|
| **硬件平台** | 正点原子阿尔法 Linux 开发板（I.MX6ULL） |
| **开发框架** | Qt 5.12 + C++ |
| **核心功能** | 人脸识别门禁 + 实时监控 + 多模态认证 |
| **应用场景** | 家庭/办公室门禁系统原型 |
| **技术难度** | 中等偏高，涉及 Linux 驱动开发、Qt GUI、网络通信、人脸识别、硬件联动 |

### 1.2 系统整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      局域网（LAN）                        │
│                                                         │
│  ┌──────────────────────┐      ┌──────────────────────┐ │
│  │     客户端 (Client)    │      │    服务端 (Server)     │ │
│  │  Linux 桌面 PC        │      │  I.MX6U 开发板        │ │
│  │                      │      │                      │ │
│  │  - Qt GUI 界面       │ UDP  │  - 摄像头视频采集      │ │
│  │  - 视频监控显示      │◄────►│  - UDP 广播视频流      │ │
│  │  - 人脸识别(Dlib)    │      │  - TCP 命令接收       │ │
│  │  - 人脸库管理(SQLite) │      │  - 硬件外设控制        │ │
│  │  - 远程开锁控制       │ TCP  │  - 门禁状态检测        │ │
│  │  - 密码修改(AES)     │◄────►│                      │ │
│  │                      │      │                      │ │
│  └──────────────────────┘      └──────────────────────┘ │
│                              │                          │
│                              ├── SPI → RC522 (RFID)     │
│                              ├── I2C → AT24C64 (EEPROM) │
│                              ├── I2C → ADS1115 (ADC)    │
│                              ├── I2C → MPU6050 (陀螺仪)  │
│                              ├── PWM → SG90 (舵机)       │
│                              └── DVP → OV2640 (摄像头)   │
└─────────────────────────────────────────────────────────┘
```

### 1.3 双通道通信机制

**通道一：UDP 视频流（端口 8888）**

```
服务端（视频采集线程）:
  摄像头 → RGB565帧 → JPEG编码 → Base64编码 → UDP广播(255.255.255.255:8888)

客户端（UDP接收）:
  UDP数据包 → Base64解码 → JPEG解码 → QImage → cv::Mat → 人脸识别/界面显示
```

**通道二：TCP 控制命令（端口 8878）**

客户端发送文本命令到服务端，服务端解析后执行对应操作：

| 命令格式 | 功能 |
|----------|------|
| `unlock` | 远程开锁（舵机转动90°→3秒后自动复位） |
| `StartVideo` | 启动摄像头采集（红外触发模式） |
| `CloseVideo` | 关闭摄像头采集 |
| `notify_msg:消息内容:时长` | 在服务端OLED屏幕显示通知 |
| `set_pwd:Base64加密数据` | 修改服务端密码（AES-128加密传输） |
| `WriteCard:用户ID:姓名` | 向空白RFID卡写入用户信息 |
| `SyncUsers:id1\|name1,id2\|name2` | 同步客户端人脸库到服务端EEPROM |

## 二、客户端（video_client）详细分析

### 2.1 客户端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Qt | 5.12 | GUI框架、网络通信 |
| C++ | C++17 | 核心逻辑 |
| OpenCV | 3.1+ | 图像处理 |
| Dlib | 19.24 | 人脸识别（HOG检测 + ResNet特征提取） |
| libfacedetection | - | 备用CNN人脸检测 |
| SQLite | 3.22 | 人脸特征数据库 |
| AES-128 | 自实现 | 密码加密传输 |

### 2.2 核心模块

#### 2.2.1 主界面（MainWindow）

- 视频画面显示（QLabel + QImage）
- 人脸识别结果叠加绘制（绿色框 + 姓名）
- 底部控制按钮：远程开锁、通知发送、修改密码、IC卡绑定、人脸库管理
- 连接状态指示

#### 2.2.2 人脸识别流水线（重点）

这是整个项目最核心的 AI 部分，流程如下：

```
UDP接收图像帧
    │
    ▼
Base64解码 → JPEG解码 → cv::Mat
    │
    ▼
FaceService::processFrame()
    │
    ▼
FaceRecognizerThread（独立线程，生产者-消费者模式）
    │
    ├── 采样间隔：每2帧处理一次（sampleInterval=2）
    │
    ├── Step 1: Dlib HOG+SVM 人脸检测
    │       → 返回人脸矩形框位置
    │
    ├── Step 2: 68关键点定位（shape_predictor_68_face_landmarks.dat）
    │       → 返回68个面部特征点坐标
    │
    ├── Step 3: ResNet特征提取（dlib_face_recognition_resnet_model_v1.dat）
    │       → 生成128维特征向量（descriptor）
    │
    ├── Step 4: 欧氏距离匹配（阈值0.6）
    │       → 遍历SQLite数据库中所有已注册人脸
    │       → distance < 0.6 → 匹配成功
    │
    └── Step 5: 防抖处理（3秒冷却期）
            → 匹配成功 → 发送"unlock"命令
```

**关键性能优化：**
- 识别在独立线程中运行，不阻塞GUI
- 生产者-消费者模式：QWaitCondition 控制帧队列
- 跳帧采样：不是每帧都识别，减少计算量
- 识别冷却期：避免重复开锁

#### 2.2.3 网络管理（NetworkManager）

```
TCP长连接管理:
  - 连接建立/断开/错误处理
  - 心跳机制（定时发送keepalive）
  - 断线自动重连
  - 命令发送队列

UDP视频接收:
  - QUdpSocket 绑定端口8888
  - 接收数据帧 → 信号通知FaceService处理
```

#### 2.2.4 人脸库管理（FaceDbManager）

**SQLite数据库文件：`face_db.db`**

表结构：
```sql
face_registry (
    id           TEXT PRIMARY KEY,  -- UUID生成的唯一ID
    name         TEXT,              -- 用户姓名
    identity     TEXT,              -- 身份信息
    descriptor   BLOB,              -- 128维特征向量（QDataStream序列化）
    preview      BLOB,              -- JPEG预览图
    create_time  TIMESTAMP          -- 注册时间
)
```

操作流程：
1. **注册人脸**：拍摄照片 → Dlib检测 → 提取特征 → 存入数据库
2. **删除人脸**：按ID删除记录
3. **加载数据库**：启动时将所有 descriptor 加载到内存（加速匹配）

#### 2.2.5 UI管理（UIManager）

- 通知弹窗（NotifyDialog）：显示服务端返回的状态消息
- 密码修改对话框（ChangePwdDialog）：旧密码验证 + 新密码设置
- IC卡写入对话框（FaceDbDialog）：绑定用户到RFID卡

### 2.3 安全机制

| 机制 | 实现方式 | 弱点 |
|------|---------|------|
| 密码传输 | AES-128-ECB加密 | 密钥硬编码在源码中 |
| 密码存储 | SHA-256哈希 + 固定盐值 | 盐值也硬编码 |
| 人脸识别 | 欧氏距离阈值0.6 | 无活体检测，可能被照片欺骗 |

## 三、服务端（video_server）详细分析

### 3.1 服务端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Qt | 5.12 | 事件驱动、网络服务 |
| OpenCV | 3.1 | 图像编码（JPEG） |
| ARM交叉编译 | Cortex-A7 | 目标平台 |
| V4L2 | Linux内核 | 摄像头驱动 |

### 3.2 核心模块

#### 3.2.1 视频采集线程（CaptureThread）

```
V4L2设备 (/dev/video1)
    │
    ├── 初始化：设置格式为 V4L2_PIX_FMT_RGB565，640×480
    ├── mmap映射：3个循环缓冲区
    │
    └── 采集循环：
        ├── 从队列获取已填充缓冲区
        ├── RGB565 → BGR转换（OpenCV）
        ├── BGR → JPEG编码
        ├── JPEG → Base64编码
        ├── UDP广播到 255.255.255.255:8888
        └── 将缓冲区放回队列
```

**注意**：此模块使用 `#ifdef __arm__` 条件编译，仅在ARM平台编译。

#### 3.2.2 TCP服务器（TcpServerManager）

- 监听 0.0.0.0:8878
- 支持多客户端同时连接
- 解析文本命令并执行
- 命令执行后向客户端回传状态

#### 3.2.3 硬件管理（HardwareManager）

统一管理所有外设，是服务端最复杂的模块：

```
HardwareManager
    │
    ├── CaptureThread      → 摄像头视频采集推流
    ├── Rc522              → RFID读卡器（100ms轮询）
    ├── At24c64            → EEPROM数据存储
    ├── ServoControl       → 舵机门锁控制
    ├── AdcCaptureThread   → ADC门状态检测
    ├── Mpu6050Thread      → 振动检测
    ├── NotificationService → 通知显示
    └── PasswordDialog     → 密码验证界面
```

#### 3.2.4 EEPROM存储方案（AT24C64）

**内存布局（8KB容量）：**

```
地址偏移    内容              大小
0x0000      EEP_Head          32字节
  ├─ magic       魔数 0x5A5A   2字节
  ├─ user_count  用户数量      2字节
  └─ reserved    保留          28字节

0x0020      EEP_Key           32字节
  ├─ aes_key     AES密钥      16字节
  └─ reserved    保留          16字节

0x0040      EEP_Pwd           32字节
  └─ sha256_hash SHA-256哈希  32字节

0x0060      EEP_Param         32字节
  ├─ unlock_mode 开锁模式      1字节
  ├─ beep_enable  蜂鸣器开关   1字节
  └─ reserved     保留         30字节

0x0080+     EEP_User[]        每用户32字节
  ├─ name        姓名          16字节
  ├─ id          用户ID        4字节
  └─ reserved    保留          12字节
```

**容量**：最多存储约 248 个用户（(8192 - 0x0080) / 32 ≈ 248）。

#### 3.2.5 舵机控制（ServoControl）

```
SG90舵机 → PWM控制
    │
    ├── 0°  → 门锁闭合（舵机水平位置）
    ├── 90° → 门锁打开（舵机垂直位置）
    │
    └── 自动关锁：
        ├── 开锁后启动5秒定时器
        ├── 检查ADC门状态
        ├── 门关闭 → 3秒后舵机回0°
        └── 门未关闭 → 不关锁（防夹/防尾随）
```

#### 3.2.6 ADC门状态检测（ADS1115）

```
ADS1115 (16位ADC，I2C接口)
    │
    ├── 通道0：门磁检测（门开/关状态）
    ├── 通道1：人体接近感应（红外）
    ├── 通道2：物理旋钮位置检测
    │
    └── 用途：
        ├── 人体靠近 → 触发人脸检测
        ├── 门状态 → 决定是否可以关锁
        └── 旋钮 → 物理开锁方式
```

#### 3.2.7 MPU6050振动检测

```
MPU6050 (六轴陀螺仪，I2C接口)
    │
    ├── 采样频率：200ms一次（5Hz）
    ├── 告警算法：
    │     合加速度 = |ax|+|ay|+|az|-1.0g
    │     合角速度 = |gx|+|gy|+|gz|
    │     阈值：加速度>12.5g 或 角速度>500°/s
    │
    └── 联动：仅在门锁闭时启用（门开着不告警）
```

#### 3.2.8 RFID读卡（RC522）

```
RC522模块 (SPI接口，ISO14443A)
    │
    ├── 100ms轮询一次
    ├── 检测卡片 → 读取块0x11数据
    ├── 解析：4字节用户ID + 16字节姓名
    ├── 查询EEPROM验证用户
    ├── 验证通过 → 开锁
    └── 写卡模式：客户端发送用户信息 → 写入空白卡
```

### 3.3 多开锁方式统一管理

系统支持5种开锁方式，统一由舵机控制模块执行：

```
开锁方式               触发源              验证逻辑
────────────────────────────────────────────────────
1. 人脸识别            客户端识别成功       TCP "unlock"命令
2. RFID刷卡            RC522读取卡片       EEPROM用户匹配
3. 密码验证            触摸屏输入          SHA-256哈希比对
4. 远程开锁            客户端按钮          TCP "unlock"命令
5. 物理旋钮            ADC通道2检测        旋钮位置判断
                                    │
                                    ▼
                          统一开锁执行：
                          ServoControl::setAngle(90)
                          → 延时 → ServoControl::setAngle(0)
```

## 四、Linux内核驱动分析

### 4.1 驱动总览

| 驱动 | 设备节点 | 接口 | 内核模块文件 |
|------|---------|------|------------|
| RC522 RFID | `/dev/rc522` | SPI | `rc522.ko` |
| MPU6050 | `/dev/my_mpu6050` | I2C | `my_6050.ko` |
| ADS1115 | `/dev/my_ads1115` | I2C | `ads1115.ko` |
| SG90舵机 | `/dev/sg90_servo` | PWM | 字符设备 |

### 4.2 RC522驱动

- **SPI字符设备驱动**
- 通过 ioctl 实现读写控制
- 支持：寻卡、防冲突、选卡、读写块数据
- 设备树匹配：`"alientek,rc522"`

### 4.3 MPU6050驱动

- **I2C字符设备驱动**
- 初始化：唤醒、配置量程（±2g，±250°/s）
- read() 返回：6字节加速度 + 6字节角速度原始数据
- 设备树匹配：`"my_mpu6050"`

### 4.4 ADS1115驱动

- **I2C字符设备驱动**
- 支持4通道单端输入
- ioctl 选择通道，read 返回16位ADC值
- 设备树匹配：`"ti,ads1115"`

### 4.5 SG90舵机驱动

- 通过 Linux PWM 子系统实现
- 字符设备接口，ioctl 设置PWM占空比
- 对应角度：0°（锁）到 90°（开）

## 五、项目存在的问题与改进方向

### 5.1 安全性问题

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| AES密钥硬编码 | 高 | 客户端源码中可见，反编译即可获取 |
| SHA-256盐值硬编码 | 高 | 同上 |
| AES-ECB模式 | 中 | 相同明文产生相同密文，应使用CBC/GCM |
| RFID使用默认密钥 | 中 | Mifare默认全FF密钥，可被轻易复制 |
| TCP明文传输 | 中 | 除密码外的命令未加密 |
| 无活体检测 | 中 | 人脸识别可能被照片欺骗 |

### 5.2 代码质量问题

| 问题 | 说明 |
|------|------|
| 编译路径硬编码 | OpenCV路径写死为`/home/tom/tool/opencv-3.1.0/` |
| 内核路径硬编码 | 驱动Makefile中`/home/tom/linux` |
| unreachable code | servocontrol.cpp中有不可达的return语句 |
| 密码明文显示 | 修改密码对话框中密码未隐藏 |
| 包含.c文件 | mainwindow.cpp中`#include "aes.c"`不规范 |

### 5.3 架构限制

| 限制 | 说明 |
|------|------|
| 服务端视频仅ARM可用 | `#ifdef __arm__` 限制了跨平台编译 |
| EEPROM容量有限 | 最多248个用户，无法扩展 |
| 无云端同步 | 人脸库仅在本地SQLite |
| 无移动端支持 | 只能通过桌面客户端操作 |

## 六、总结

VisionPass 是一个功能完善的嵌入式门禁系统原型，涵盖了：

- **嵌入式Linux驱动开发**（I2C/SPI/PWM/V4L2）
- **Qt跨平台应用开发**（GUI/网络/多线程/数据库）
- **AI人脸识别**（Dlib深度学习模型）
- **网络通信**（TCP/UDP自定义协议）
- **硬件联动**（多传感器协同工作）
- **安全设计**（加密/哈希/分区存储）

作为学习项目，它覆盖了嵌入式AI方向的大量核心技能，是一个非常有价值的实战项目。
