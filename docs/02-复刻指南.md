# VisionPass 复刻指南（嵌入式Linux入门向）

> **目标读者**：嵌入式Linux初学者，目标是从事嵌入式AI方向
> **硬件要求**：正点原子阿尔法（I.MX6ULL）开发板 + 摄像头 + RC522 + SG90舵机 + MPU6050 + ADS1115 + AT24C64

## 阶段二：硬件连接与驱动开发（1-2周）

> **学习重点**：这个阶段是嵌入式Linux核心技能，建议自己动手写驱动，不要直接复制原作者的代码。

### 2.1 硬件接线图

```
I.MX6U 开发板
    │
    ├── I2C1 (SCL/SDA)
    │     ├── ADS1115 (地址0x48) ── 门磁检测 + 人体感应 + 旋钮
    │     ├── MPU6050 (地址0x68) ── 振动检测
    │     └── AT24C64 (地址0x50) ── EEPROM存储
    │
    ├── SPI3 (MISO/MOSI/SCLK/CS)
    │     └── RC522 ── RFID读卡
    │
    ├── PWM3
    │     └── SG90舵机 ── 门锁控制
    │
    ├── CSI (DVP接口)
    │     └── OV2640摄像头 ── 视频采集
    │
    └── GPIO
          └── 红外人体传感器 ── 触发信号
```

### 2.2 设备树配置（第一步）

在正点原子的设备树文件（通常是 `arch/arm/boot/dts/imx6ull-14x14-evk.dts`）中添加：

```dts
/* 1. 使能I2C1 */
&i2c1 {
    clock-frequency = <100000>;
    pinctrl-names = "default";
    pinctrl-0 = <&pinctrl_i2c1>;
    status = "okay";

    /* ADS1115 */
    ads1115@48 {
        compatible = "ti,ads1115";
        reg = <0x48>;
    };

    /* MPU6050 */
    my_mpu6050@68 {
        compatible = "my_mpu6050";
        reg = <0x68>;
    };

    /* AT24C64 - 使用内核自带的at24驱动 */
    eeprom@50 {
        compatible = "atmel,24c64";
        reg = <0x50>;
        pagesize = <32>;
    };
};

/* 2. 使能SPI3 */
&ecspi3 {
    fsl,spi-num-chipselects = <1>;
    cs-gpio = <&gpio1 20 GPIO_ACTIVE_LOW>;
    pinctrl-names = "default";
    pinctrl-0 = <&pinctrl_ecspi3>;
    status = "okay";

    rc522@0 {
        compatible = "alientek,rc522";
        spi-max-frequency = <8000000>;
        reg = <0>;
    };
};

/* 3. 使能PWM3 */
&pwm3 {
    pinctrl-names = "default";
    pinctrl-0 = <&pinctrl_pwm3>;
    status = "okay";
};

/* 4. 使能CSI摄像头 */
&csi {
    status = "okay";
};
```

**编译并替换设备树**：
```bash
# 在Linux内核源码目录
make dtbs
# 将生成的 .dtb 文件复制到开发板启动分区
# 重启开发板
```

### 2.3 驱动开发顺序（由简单到难）

#### 2.3.1 第一个驱动：SG90舵机（PWM字符设备）

这是最简单的驱动，适合入门：

```c
// 伪代码框架，建议自己实现
#include <linux/module.h>
#include <linux/fs.h>
#include <linux/pwm.h>
#include <linux/miscdevice.h>

static struct pwm_device *pwm;
static int sg90_ioctl(struct file *filp, unsigned int cmd, unsigned long arg) {
    // cmd: 角度值 (0-90)
    // 将角度转换为PWM占空比
    // pwm_config(pwm, duty_ns, period_ns)
    return 0;
}

static struct file_operations sg90_fops = {
    .owner   = THIS_MODULE,
    .unlocked_ioctl = sg90_ioctl,
};

static struct miscdevice sg90_misc = {
    .minor = MISC_DYNAMIC_MINOR,
    .name  = "sg90_servo",
    .fops  = &sg90_fops,
};

static int __init sg90_init(void) {
    pwm = pwm_get(NULL, "pwm3");
    pwm_enable(pwm);
    return misc_register(&sg90_misc);
}

module_init(sg90_init);
MODULE_LICENSE("GPL");
```

**测试方法**：
```bash
insmod sg90.ko
echo 90 > /dev/sg90_servo  # 应该转90度
echo 0 > /dev/sg90_servo   # 应该回0度
```

### 3.2 开发顺序（推荐）

**建议按以下顺序逐个模块开发**，每完成一个就测试一个：

```
Step 1: 主窗口 (mainwindow)
    → 创建基本界面，确认Qt for ARM能正常运行

Step 2: 舵机控制 (servocontrol)
    → 测试PWM驱动，确认能控制舵机转动

Step 3: EEPROM读写 (at24c64)
    → 使用内核自带的at24驱动，通过/dev/i2c-0或sysfs读写
    → 实现用户信息存储/读取

Step 4: TCP服务器 (tcpservermanager)
    → 实现监听端口、接收命令、返回状态
    → 先用简单的"unlock"命令测试

Step 5: 密码验证 (passworddialog)
    → 实现虚拟键盘界面
    → 实现SHA-256哈希计算和比对

Step 6: 视频采集 (capture_thread)
    → V4L2初始化、mmap映射
    → RGB565转JPEG、Base64编码、UDP广播
    → 这是性能关键模块

Step 7: RFID读卡 (rc522)
    → 轮询读取卡片
    → EEPROM验证

Step 8: ADC检测 (adccapture_thread)
    → 门状态、人体感应、旋钮

Step 9: MPU6050振动 (mpu6050_thread)
    → 定时采样、阈值判断

Step 10: 通知服务 (notificationservice)
    → OLED屏幕显示
```

### 3.3 关键代码框架

**主窗口启动流程**：
```cpp
// mainwindow.cpp 伪代码
void MainWindow::init() {
    // 1. 初始化硬件管理器
    hardwareMgr = new HardwareManager(this);

    // 2. 初始化TCP服务器
    tcpServer = new TcpServerManager(this);
    connect(tcpServer, &TcpServerManager::unlockRequest,
            hardwareMgr, &HardwareManager::unlock);
    connect(tcpServer, &TcpServerManager::notifyRequest,
            this, &MainWindow::showNotification);

    // 3. 启动TCP监听
    tcpServer->start(8878);

    // 4. 初始化舵机（默认锁门状态）
    hardwareMgr->initServo();
    hardwareMgr->lockDoor();
}
```

```cpp
    fmt.fmt.pix.width = 640;

    ioctl(fd, VIDIOC_S_FMT, &fmt);

    while (!stopped) {

        // RGB565 -> BGR (OpenCV cvtColor)
        // BGR -> JPEG (imencode)
        // JPEG -> Base64
        // UDP发送
        // 入队空缓冲区
    }

    close(fd);
}
```

### 5.2 常见问题排查

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 摄像头打不开 | 设备树未配置/驱动未加载 | 检查 `/dev/video1` 是否存在 |
| 视频帧率极低 | OpenCV未开启NEON优化 | 重新编译OpenCV，加 `-DENABLE_NEON=ON` |
| I2C设备找不到 | 设备树引脚配置错误 | 用 `i2cdetect -y 0` 检查 |
| SPI通信失败 | 片选引脚未正确配置 | 检查设备树SPI配置 |
| 人脸识别太慢 | 未使用独立线程 | 确认FaceRecognizerThread在运行 |
| TCP连接被拒绝 | 防火墙/端口未监听 | `netstat -tlnp` 检查服务端 |
| 舵机不转 | PWM未使能/引脚复用 | 检查 `sysfs/class/pwm/` |

## 阶段六：拓展与改进方向

> 以下项目按难度递增排列，你可以选择感兴趣的来做。每个项目都标注了适合的阶段和对嵌入式AI方向的价值。

### 6.1 入门级拓展（建议先做）

#### A. 添加活体检测（防照片欺骗）
- **难度**：★★
- **技术方案**：
  - 简单版：眨眼检测（检测眼睛关键点变化）
  - 进阶版：基于深度学习的活体检测模型（如MiniFAS）
- **对嵌入式AI的价值**：学习模型在嵌入式端的部署和优化
- **涉及知识**：OpenCV眼部ROI提取、TFLite/NCNN模型推理

#### B. 替换人脸识别模型为轻量级方案
- **难度**：★★
- **技术方案**：使用 MobileFaceNet 或 ArcFace 的量化版本
- **工具**：NCNN / TNN / MNN（腾讯/阿里的嵌入式推理框架）
- **对嵌入式AI的价值**：学习模型转换、量化、嵌入式推理部署
- **为什么需要**：Dlib的ResNet-34在ARM上运行较慢，轻量模型可以显著提升速度

#### C. 添加日志系统
- **难度**：★
- **技术方案**：spdlog 库，记录所有开锁事件、告警信息
- **对嵌入式AI的价值**：为后续AI行为分析积累数据

#### D. 添加Web管理界面
- **难度**：★★
- **技术方案**：在开发板上运行轻量HTTP服务器（如mongoose），提供REST API
- **价值**：不用桌面客户端也能管理设备

### 6.2 中级拓展

#### E. 使用NPU/GPU加速推理
- **难度**：★★★
- **背景**：I.MX6U没有NPU，但如果你升级到 I.MX8M Plus（有NPU），可以：
  - 使用NXP eIQ工具链
  - 将模型转换为TFLite格式
  - 在NPU上运行推理
- **对嵌入式AI的价值**：这是工业界嵌入式AI的主流方案
- **学习路径**：NPU原理 → TFLite → eIQ工具链 → 模型量化

#### F. MQTT云端集成
- **难度**：★★
- **技术方案**：
  - 开发板运行MQTT客户端（Paho C库）
  - 连接自建EMQX或公有云MQTT Broker
  - 上传开锁日志、告警信息、设备状态
- **价值**：学习IoT设备与云端通信

#### G. 语音交互
- **难度**：★★★
- **技术方案**：
  - 添加麦克风模块（如I2S接口的INMP441）
  - 离线语音识别（使用Porcupine或Sherpa-ONNX）
  - 语音指令开锁："开门"、"关门"
- **对嵌入式AI的价值**：学习嵌入式语音AI处理

#### H. 添加摄像头自动对焦和人脸追踪
- **难度**：★★★
- **技术方案**：
  - 使用舵机云台搭载摄像头
  - 基于人脸检测框中心控制云台转动
  - PID算法实现平滑追踪
- **对嵌入式AI的价值**：学习视觉+控制的闭环系统

### 6.3 高级拓展（嵌入式AI核心方向）

#### I. 端侧AI完整方案：模型训练→量化→部署
- **难度**：★★★★
- **完整流程**：
  1. 在PC上采集人脸数据，训练一个轻量级人脸识别模型（MobileFaceNet）
  2. 使用TensorFlow/PyTorch训练
  3. 模型量化（INT8或FP16）
  4. 转换为TFLite/ONNX格式
  5. 使用NCNN/MNN在开发板上部署
  6. 对比与原Dlib方案的速度和准确率
- **对嵌入式AI的价值**：这是最贴近工业实践的完整链路

#### J. 多模态融合识别
- **难度**：★★★★
- **技术方案**：
  - 人脸 + 声纹（说话人识别）双重认证
  - 两个模型都在嵌入式端运行
  - 决策融合策略
- **对嵌入式AI的价值**：学习多模态AI在嵌入式的部署

#### K. 边缘计算：在开发板上运行完整的YOLO目标检测
- **难度**：★★★★
- **技术方案**：
  - 使用YOLOv5/v8 Nano版本
  - NCNN部署
  - 实现：人员检测 + 人脸识别联动
  - 只有检测到人员时才触发人脸识别
- **对嵌入式AI的价值**：YOLO是嵌入式AI最常见的应用场景

#### L. TinyML：超低功耗AI方案
- **难度**：★★★★★
- **背景**：如果你有一块STM32开发板（带MCU）
  - 使用TensorFlow Lite for Microcontrollers
  - 在MCU上运行极小的人脸检测模型
  - 仅当检测到人脸时才唤醒I.MX6U
- **对嵌入式AI的价值**：TinyML是嵌入式AI的前沿方向

### 6.4 工程化改进

#### M. 修复安全问题
- 将AES密钥从硬编码改为动态生成（首次启动时）
- 使用AES-CBC或AES-GCM替代ECB模式
- 添加TLS加密TCP通信
- RFID使用非默认密钥

#### N. 构建系统改进
- 使用CMake替代qmake（更现代化）
- 添加CI/CD（GitHub Actions自动编译）
- Docker容器化客户端

#### O. 升级开发板
- I.MX6U → I.MX8M Plus（4核A53 + NPU，更适合AI）
- 或者使用Rockchip RK3588（6TOPS NPU，性价比极高）
- 或者使用瑞芯微RV1106/1103（专为AI IPC设计，内置NPU）

## 时间规划建议

```
第1-2周：  知识储备 + 环境搭建
第3-4周：  驱动开发 + 硬件联调
第5-6周：  服务端开发 + 部署到开发板
第7-8周：  客户端开发 + 联调测试
第9-10周： 拓展改进（选1-2个做）
```

**重要建议**：
1. 不要试图一次性全部完成，按阶段推进
2. 每个阶段完成后做一次总结，记录遇到的问题
3. 驱动开发阶段多使用正点原子的示例代码作为参考
4. 人脸识别阶段先在PC上跑通，再考虑移植到开发板
5. 遇到问题先查日志（dmesg、串口输出），再问AI或搜索
