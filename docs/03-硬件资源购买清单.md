# VisionPass AI门禁系统 — 硬件资源清单（最终版）

> **对比**：原版VisionPass（正点原子Mini板 + ADS1115 + EEPROM）
> **你的方案**：正点原子阿尔法板 + 板载AP3216C + GPIO门磁 + 文件存储

---

## 一、代码级确认：原版到底用了什么

通过分析原作者的 `adccapture_thread.cpp` 和 `mainwindow.cpp`，确认了三个ADC通道的实际用途：

| ADC通道 | 代码中的处理函数 | 实际用途 | 阈值 | 触发效果 |
|---------|-----------------|---------|------|---------|
| **通道0** (`adc_val[0]`) | `handleChannel1()` | **人体靠近检测** | value > 20000 | 启动摄像头采集 + 开启本地显示 |
| **通道1** (`adc_val[1]`) | `handleChannel2()` | **门是否关上检测** | value > 20000 | 允许舵机自动复位关锁 |
| **通道2** (`adc_val[2]`) | `processAdcValue()` / `handleChannel3()` | 旋钮位置（空实现） | 0-500/500-2500 | 物理开锁（实际未完整实现） |

**关键发现**：

1. **原作者没有用专门的门磁模块**，而是用一个红外/距离传感器的模拟输出接到ADS1115通道1上，通过电压值判断门是否关上。门关上时红外传感器被遮挡，ADC值变高。

2. **人体检测用的也是一个红外传感器的模拟输出**，接到ADS1115通道0上。

3. **通道3（旋钮）的handler是空的**（`handleChannel3`函数体为空），说明原作者计划做但没做完。

4. **整个ADS1115的作用就是读两个红外传感器的模拟电压值**。你的板载AP3216C已经是I2C数字传感器了，直接读寄存器就能得到"有人/无人"状态，比原版更简单。

### AP3216C 详细说明

你的板载 AP3216C 是一个三合一传感器（I2C地址 0x1e），已经焊在板子上：

| 功能 | 输出方式 | 用途 |
|------|---------|------|
| 环境光照度 | 数字（lux值） | 可选：光线暗时降低识别频率 |
| **红外接近检测** | 数字（有/无） | **替代原版人体检测** |
| 距离检测 | 数字（距离值） | **替代原版门检测** |

**和原版的关键区别**：

| | 原版（ADS1115方案） | 你的方案（AP3216C方案） |
|--|---------------------|----------------------|
| 人体检测 | 模拟电压 → ADC → 阈值判断 | I2C读寄存器 → 直接得到数字值 |
| 门检测 | 模拟电压 → ADC → 阈值判断 | I2C读寄存器 → 直接得到距离值 |
| 需要额外模块 | 是（ADS1115 + 2个红外传感器） | **否，板载即可** |
| 代码复杂度 | 需要ADC驱动 + 电压换算 | I2C直接读，代码更简单 |

## 四、不买的东西及替代方案

### 4.1 不买 ADS1115 ADC模块

| 原版用途 | 你的替代 | 实现方式 |
|---------|---------|---------|
| 人体靠近检测（通道0） | **板载AP3216C红外接近** | I2C读红外寄存器，有/无人直接得到数字值 |
| 门状态检测（通道1） | **板载AP3216C距离 + 门磁GPIO** | AP3216C可初步判断，门磁模块做精确检测 |
| 旋钮（通道2，未实现） | **板载功能按钮** | 短按=开锁，长按=其他功能 |

**为什么可以替代**：
- 原版用ADS1115是因为它的红外传感器输出的是**模拟电压**，需要ADC转换成数字
- 你的AP3216C本身就是**数字传感器**，I2C读出来就是处理好的数值
- 代码上从"读ADC设备 → 换算电压 → 阈值判断"变成"读I2C寄存器 → 判断数值"，反而更简单

### 4.2 不买 AT24C64 EEPROM

| 原版存储内容 | 你的替代方案 | 文件路径示例 |
|-------------|------------|------------|
| 用户信息（ID+姓名） | SQLite数据库 | `/opt/visionpass/data/users.db` |
| 密码SHA-256哈希 | JSON配置文件 | `/opt/visionpass/config/system.json` |
| AES密钥 | JSON配置文件（首次启动随机生成） | `/opt/visionpass/config/system.json` |
| 系统参数（IP/端口等） | JSON配置文件 | `/opt/visionpass/config/system.json` |
| RFID用户映射 | SQLite数据库 | `/opt/visionpass/data/users.db` |
| 人脸特征descriptor | SQLite数据库 | `/opt/visionpass/data/users.db` |

**示例 system.json**：
```json
{
    "password_hash": "a1b2c3d4e5f6...",
    "aes_key": "0123456789abcdef0123456789abcdef",
    "server_ip": "127.0.0.1",
    "tcp_port": 8878,
    "face_threshold": 0.6,
    "auto_lock_delay_ms": 5000
}
```

**为什么可以替代**：
- 你的开发板有eMMC或SD卡，文件系统就是掉电不丢失的
- 原版用EEPROM是因为Mini板可能没有大容量存储
- 文件存储反而更灵活（JSON可读可编辑，SQLite支持查询）

## 五、接口映射与接线

### 接线总览

```
阿尔法开发板
│
├── CSI接口 ────────────────── OV2640摄像头（必买#1）
│
├── ECSPI4引脚 ─────────────── RC522 RFID模块（必买#2）
│   ├── MISO  → UART2_RTS_B
│   ├── MOSI  → UART2_CTS_B
│   ├── SCLK  → UART2_RX_DATA
│   └── CS    → GPIO1_IO20（软件GPIO片选）
│
├── PWM3引脚 ───────────────── SG90舵机（必买#3）
│   └── 信号线 → GPIO1_IO04（复用为PWM3_OUT）
│
├── GPIO引脚 ───────────────── 门磁开关模块（必买#4）
│   └── 信号线 → 任意空闲GPIO（如GPIO1_IO03）
│
├── 板载AP3216C ────────────── 人体检测 + 距离检测（已有）
│   └── I2C1总线，地址0x1e，直接读寄存器
│
├── 板载ICM20608 ───────────── 振动检测（已有）
│   └── ECSPI3总线
│
├── 板载WM8960 ────────────── 语音解锁（已有）
│   ├── 麦克风 → 录音识别"开门"
│   └── 扬声器 → 语音反馈
│
├── 板载蜂鸣器 ────────────── 开锁提示音（已有）
│
├── 板载功能按钮 ───────────── 物理开锁（已有）
│
└── 板载AP3216C红外 ───────── 人体靠近触发（已有）
    └── 替代原版ADS1115通道0
```

### 门磁模块接线

门磁开关模块是**数字输出**（干簧管），接线极其简单：

```
门磁模块              开发板
┌─────────┐
│  VCC    │──────── 3.3V排针
│  GND    │──────── GND排针
│  OUT    │──────── GPIO1_IO03（任意空闲GPIO）
└─────────┘

门关上 → 磁铁靠近 → OUT输出低电平（0）
门打开 → 磁铁远离 → OUT输出高电平（1）

代码中读取：
int doorClosed = gpio_read_value(GPIO1_IO03);  // 0=关, 1=开
```

## 六、设备树需要添加的节点

| 节点 | 说明 | 状态 |
|------|------|------|
| `&ecspi4` | RC522 RFID | **需要添加** |
| `&pwm3` | SG90舵机 | **需要添加** |
| GPIO门磁 | 用gpio-keys或自定义驱动 | **需要添加** |
| AP3216C | 已在设备树中（`ap3216c@1e`） | **已有，不需要改** |
| ICM20608 | 已在设备树中（`icm20608@0`） | **已有，不需要改** |
| WM8960 | 已在设备树中（`wm8960@1a`） | **已有，不需要改** |
| OV2640 | 已在设备树中（`ov2640@0x30`） | **已有，不需要改** |

**注意**：你的官方设备树已经配置了OV2640、AP3216C、ICM20608、WM8960，只需要额外添加RC522（SPI4）、PWM3、GPIO门磁三个节点。

4. **杜邦线**：母对母和母对公各买2份以上，接线时很容易不够
