## 🎓 学习路径（从零开始）

### 第一阶段：基础知识储备（建议 2-3 周）

#### 1.1 Linux 基础命令
**为什么要学：** 所有操作都在 Linux 终端进行

**需要掌握：**
```bash
# 文件操作
ls, cd, pwd, mkdir, rm, cp, mv, cat, head, tail

# 权限管理
chmod, chown, sudo

# 进程管理
ps, top, kill, &

# 网络配置
ifconfig, ping, ssh, scp

# 文本处理
grep, find, sed, awk
```

**推荐资源：**
- 《鸟哥的Linux私房菜》基础篇（前 10 章）
- Linux 命令行在线练习：https://linuxcommand.org/

#### 1.2 C/C++ 编程基础
**为什么要学：** 项目核心代码都是 C++

**需要掌握：**
```cpp
// 基础语法
变量、函数、条件语句、循环、数组、指针

// 面向对象
类、继承、多态、构造函数、析构函数

// 内存管理
new/delete、智能指针（项目未用，但需了解）
```

**推荐资源：**
- 《C++ Primer》第 5 版（前 15 章）
- 在线练习：https://www.learncpp.com/

### 第二阶段：嵌入式 Linux 专项（建议 2-3 周）

#### 2.1 交叉编译
**为什么要学：** 在 PC 上编译，在开发板上运行

**学习内容：**
```bash
# 理解交叉编译的概念
主机架构：x86_64 (PC)
目标架构：ARM (开发板)

# 使用交叉编译工具链
arm-linux-gnueabihf-gcc hello.c -o hello

# CMake 交叉编译
cmake -DCMAKE_TOOLCHAIN_FILE=toolchain.cmake ..
make
```

**实践步骤：**
1. 编写简单的 "Hello World" 程序
2. 用交叉编译器编译
3. 用 `file` 命令检查生成的可执行文件架构
4. 通过 NFS/SCP 传到开发板运行

#### 2.2 设备树（Device Tree）
**为什么要学：** 配置硬件引脚复用

**学习内容：**
```dts
// 设备树语法
&ecspi3 {
    status = "okay";
    
    rc522@0 {
        compatible = "visionpass,rc522";
        reg = <0>;
        spi-max-frequency = <1000000>;
    };
};
```

**实践步骤：**
1. 阅读 `imx6ull-14x14-evk.dts` 文件
2. 理解 pinctrl（引脚复用配置）
3. 理解 SPI、GPIO、PWM 节点
4. 编译设备树：`make dtbs`

**推荐资源：**
- 《嵌入式Linux设备树详解》
- 内核文档：`Documentation/devicetree/`

#### 2.3 内核驱动基础
**为什么要学：** RC522、舵机、IR 传感器都需要驱动

**学习内容：**
```c
// 字符设备驱动框架
#include <linux/module.h>
#include <linux/fs.h>
#include <linux/cdev.h>

static int my_open(struct inode *inode, struct file *filp) {
    // 打开设备
    return 0;
}

static ssize_t my_read(struct file *filp, char __user *buf, 
                       size_t count, loff_t *ppos) {
    // 读取数据
    return count;
}

static struct file_operations my_fops = {
    .owner = THIS_MODULE,
    .open = my_open,
    .read = my_read,
};
```

**实践步骤：**
1. 阅读 `drivers/rc522/rc522.c` 代码
2. 理解 `file_operations` 结构体
3. 理解 `ioctl` 系统调用
4. 编译并加载驱动：`insmod rc522.ko`

**推荐资源：**
- 《Linux设备驱动程序》第 3 版
- 内核源码：`drivers/char/` 目录下的示例

#### 3.2 代码阅读技巧

**技巧 1：从 main() 函数开始**
```cpp
// app/main.cpp
int main(int argc, char *argv[])
{
    QApplication app(argc, argv);
    
    SystemController controller;  // 核心控制器
    controller.initialize();      // 初始化所有模块
    
    MainWindow window(&controller);  // 主窗口
    window.show();
    
    return app.exec();  // 进入事件循环
}
```

**技巧 2：跟踪信号槽连接**
```cpp
// 在 SystemController::initialize() 中
connect(m_faceProcessThread, &FaceProcessThread::faceRecognized,
        this, &SystemController::onFaceRecognized);
```
→ 找到 `onFaceRecognized()` 函数看做了什么

**技巧 3：理解数据流**
```
摄像头 → V4L2CaptureThread → QImage
                              ↓
                        FaceProcessThread
                              ↓
                    FaceDetector (检测人脸)
                              ↓
                    FaceRecognizer (提取特征)
                              ↓
                    UserDatabase (比对特征)
                              ↓
                    SystemController (决策)
                              ↓
                    ServoControl (开锁)
                    BeeperControl (提示音)
```

**技巧 4：使用调试输出**
```cpp
// 在关键位置添加 qDebug()

    qDebug() << "识别到用户:" << userId;  // 添加调试输出
    unlockDoor();
```

## 💡 常见问题

### Q1: 代码太多了，不知道从哪里开始看？
**A:** 从 `app/main.cpp` 开始，跟踪程序启动流程。然后看 `SystemController::initialize()`，理解各模块如何初始化。

### Q2: 看不懂 C++ 模板和 STL？
**A:** 先掌握 `vector`、`map`、`string` 三个容器即可，其他的用到时再学。

### Q3: Qt 信号槽机制不理解？
**A:** 信号槽就是"事件触发 → 执行函数"的机制。比如按钮被点击（信号）→ 执行开锁函数（槽）。

### Q4: 内核驱动代码看不懂？
**A:** 先理解字符设备驱动的基本框架（open/read/write/ioctl），然后对照 RC522 驱动代码学习。

### Q5: 编译报错怎么办？
**A:** 
1. 仔细阅读错误信息
2. 检查缺少的头文件
3. 检查函数参数类型
4. 使用 `make clean` 后重新编译
