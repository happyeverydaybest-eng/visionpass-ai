# VisionPass 故障排查手册

> 常见问题诊断与解决方案

---

## 🔍 快速诊断流程

遇到问题时，按以下流程排查：

```
程序无法启动？
    ↓
查看错误信息 → 参考"启动问题"章节
    ↓
程序启动但功能异常？
    ↓
哪个功能异常？
    ├─→ 摄像头不工作 → 参考"摄像头问题"
    ├─→ RFID 无法读卡 → 参考"RFID 问题"
    ├─→ 舵机不转动 → 参考"舵机问题"
    ├─→ IR 传感器无反应 → 参考"IR 传感器问题"
    ├─→ 人脸识别失败 → 参考"人脸识别问题"
    └─→ 程序崩溃 → 参考"崩溃问题"
```

## 1️⃣ 启动问题

### 问题 1.1：程序启动后黑屏

**现象：**
- 程序运行但屏幕无显示
- 或者屏幕显示花屏

**可能原因：**
1. Qt 平台插件未正确设置
2. 帧缓冲设备不存在
3. 分辨率不匹配
4. 显卡驱动问题

**诊断步骤：**

```bash
# 1. 检查帧缓冲设备
ls -l /dev/fb0
# 应该存在，权限为 crw-rw----

# 2. 检查 Qt 平台插件
echo $QT_QPA_PLATFORM
# 应该输出：linuxfb:fb=/dev/fb0

# 3. 查看程序输出
/opt/visionpass/bin/visionpass 2>&1
# 查看是否有错误信息
```

**解决方案：**

```bash
# 方案 1：设置正确的环境变量
export QT_QPA_PLATFORM=linuxfb:fb=/dev/fb0
export QT_QPA_GENERIC_PLUGINS=evdevkeyboard,evdevmouse

# 方案 2：指定正确的帧缓冲设备
# 如果有多个 fb 设备，尝试不同的
export QT_QPA_PLATFORM=linuxfb:fb=/dev/fb1

# 方案 3：检查分辨率
fbset -s
# 确认分辨率与程序匹配（应该是 1024x600）
```

### 问题 1.2：程序启动报错 "Cannot open /dev/xxx"

**现象：**
```
Cannot open /dev/rc522: No such file or directory
Cannot open /dev/servo: No such file or directory
Cannot open /dev/ir_sensor: No such file or directory
```

**可能原因：**
1. 驱动模块未加载
2. 设备树配置错误
3. 设备节点权限不足

**诊断步骤：**

```bash
# 1. 检查驱动是否加载
lsmod | grep -E "rc522|servo|ir_sensor"
# 应该看到对应的模块

# 2. 检查设备节点
ls -l /dev/rc522 /dev/servo /dev/ir_sensor
# 应该存在这些设备

# 3. 查看内核日志
dmesg | tail -50
# 查看驱动加载时的错误信息
```

**解决方案：**

```bash
# 方案 1：手动加载驱动
insmod /opt/visionpass/drivers/rc522.ko
insmod /opt/visionpass/drivers/servo.ko
insmod /opt/visionpass/drivers/ir_sensor.ko

# 方案 2：检查设备树
# 确认设备树中已配置这些设备
# 参考 docs/硬件接线图.md

# 方案 3：修复权限
chmod 666 /dev/rc522 /dev/servo /dev/ir_sensor
```

### 问题 1.3：程序启动报错 "Qt platform plugin could not be found"

**现象：**
```
This application failed to start because it could not find or load the Qt platform plugin "linuxfb"
```

**可能原因：**
1. Qt 插件路径未设置
2. Qt 库文件缺失
3. 环境变量错误

**诊断步骤：**

```bash
# 1. 检查 Qt 插件路径
echo $QT_PLUGIN_PATH
# 应该指向 Qt 插件目录

# 2. 查找 linuxfb 插件
find /usr -name "*linuxfb*"
# 应该找到 libqlinuxfb.so

# 3. 检查 Qt 版本
/opt/visionpass/bin/visionpass --version
# 或者查看依赖
ldd /opt/visionpass/bin/visionpass
```

**解决方案：**

```bash
# 方案 1：设置正确的插件路径
export QT_PLUGIN_PATH=/usr/lib/qt5/plugins

# 方案 2：如果找不到插件，重新安装 Qt
# 参考部署手册重新部署 Qt 库

# 方案 3：使用 strace 调试
strace -e open /opt/visionpass/bin/visionpass 2>&1 | grep linuxfb
# 查看程序在哪里查找插件
```

## 2️⃣ 摄像头问题

### 问题 2.1：摄像头无法打开

**现象：**
- 程序提示 "Cannot open camera"
- 摄像头画面黑屏

**可能原因：**
1. OV2640 排线未正确连接或方向错误
2. CSI 接口接触不良
3. 设备节点权限不足
4. 摄像头被其他程序占用

**诊断步骤：**

```bash
# 1. 检查视频设备
ls -l /dev/video*
# 应该存在 /dev/video0（TVP5150）和 /dev/video1（OV2640）

# 2. 查看内核日志
dmesg | grep -i "ov2640\|csi\|camera"
# 查看摄像头识别信息

# 3. 检查 OV2640 驱动
lsmod | grep ov2640
# 应该看到 ov2640 模块
```

**解决方案：**

```bash
# 方案 1：重新插拔 OV2640 排线，注意方向
# 方案 2：修复权限
chmod 666 /dev/video1

# 方案 3：检查摄像头是否被占用
fuser /dev/video1

# 方案 4：使用 v4l2-ctl 测试
v4l2-ctl -d /dev/video1 --list-formats
```

---

### 问题 2.2：摄像头画面卡顿

**现象：**
- 画面帧率很低（< 5fps）
- 画面延迟明显

**可能原因：**
1. CPU 负载过高
2. 摄像头分辨率设置过高
3. 图像处理过于复杂
4. 内存不足

**诊断步骤：**

```bash
# 1. 检查 CPU 使用率
top
# 查看 CPU 使用率是否接近 100%

# 2. 检查内存使用
free -m
# 查看可用内存

# 3. 检查摄像头分辨率
v4l2-ctl -d /dev/video0 --get-fmt-video
# 查看当前分辨率
```

**解决方案：**

```bash
# 方案 1：降低摄像头分辨率
# 修改 V4L2CaptureThread.cpp 中的分辨率设置
# 从 1280x720 改为 640x480

# 方案 2：降低帧率
# 修改 V4L2CaptureThread.cpp 中的 msleep(330)
# 改为 msleep(500) 降低到 2fps

# 方案 3：优化图像处理
# 减少人脸检测的频率（跳帧处理）
```

## 3️⃣ RFID 问题

### 问题 3.1：RFID 无法读卡

**现象：**
- 将卡片靠近 RC522 无反应
- 程序不显示卡片 UID

**可能原因：**
1. RC522 驱动未加载
2. SPI 接线错误
3. RC522 模块损坏
4. 卡片不兼容

**诊断步骤：**

```bash
# 1. 检查驱动加载
lsmod | grep rc522
# 应该看到 rc522 模块

# 2. 检查设备节点
ls -l /dev/rc522
# 应该存在

# 3. 查看内核日志
dmesg | grep rc522
# 查看驱动初始化信息

# 4. 手动测试读卡
# 运行测试程序（如果有）
```

**解决方案：**

```bash
# 方案 1：重新加载驱动
rmmod rc522
insmod /opt/visionpass/drivers/rc522.ko

# 方案 2：检查 SPI 接线
# 参考 docs/硬件接线图.md
# 确认 MISO/MOSI/SCK/CS 接线正确

# 方案 3：检查 RC522 供电
# 确认 VCC 接 3.3V（不是 5V！）

# 方案 4：更换卡片测试
# 确认使用 Mifare 1K 卡片
```

### 问题 3.2：RFID 读卡不稳定

**现象：**
- 有时能读到，有时读不到
- 读取距离很近

**可能原因：**
1. 天线功率不足
2. SPI 通信不稳定
3. 电源干扰
4. 卡片质量问题

**诊断步骤：**

```bash
# 1. 查看 RC522 寄存器配置
# 检查天线功率设置

# 2. 检查 SPI 通信
# 使用示波器查看 SPI 信号质量

# 3. 检查电源
# 使用万用表测量 VCC 电压
```

**解决方案：**

```bash
# 方案 1：增加天线功率
# 修改 RC522 寄存器配置
# 增加 TxControlReg 的功率设置

# 方案 2：优化 SPI 通信
# 降低 SPI 时钟频率
# 在设备树中修改 spi-max-frequency

# 方案 3：添加去耦电容
# 在 RC522 的 VCC 和 GND 之间加 100nF 电容

# 方案 4：增加读卡重试
# 修改 RFIDThread.cpp 中的重试逻辑
```

## 4️⃣ 舵机问题

### 问题 4.1：舵机不转动

**现象：**
- 识别成功后舵机无反应
- 舵机发出嗡嗡声但不转

**可能原因：**
1. 舵机驱动未加载
2. PWM 信号异常
3. 电源不足
4. 舵机损坏

**诊断步骤：**

```bash
# 1. 检查驱动加载
lsmod | grep servo
# 应该看到 servo 模块

# 2. 检查设备节点
ls -l /dev/servo
# 应该存在

# 3. 查看内核日志
dmesg | grep servo
# 查看 PWM 配置信息

# 4. 手动测试舵机
echo 90 > /dev/servo  # 转到 90 度
echo 0 > /dev/servo   # 转到 0 度
```

**解决方案：**

```bash
# 方案 1：检查电源
# 确认舵机接 5V 电源
# 测量 5V 电压是否稳定

# 方案 2：检查 PWM 信号
# 使用示波器查看 PWM 波形
# 确认频率 50Hz，脉宽 0.5-2.5ms

# 方案 3：更换舵机测试
# 确认舵机本身无问题

# 方案 4：检查接线
# 参考 docs/硬件接线图.md
# 确认信号线接 GPIO1_IO27（JP6 Pin 5）
```

---

### 问题 4.2：舵机抖动

**现象：**
- 舵机转到位置后持续抖动
- 舵机转动不平稳

**可能原因：**
1. PWM 信号不稳定
2. 电源纹波过大
3. 机械负载过大
4. 舵机齿轮磨损

**诊断步骤：**

```bash
# 1. 检查 PWM 信号
# 使用示波器查看 PWM 波形是否稳定

# 2. 检查电源
# 使用示波器查看 5V 电源纹波

# 3. 检查机械负载
# 手动转动舵机，感受阻力
```

**解决方案：**

```bash
# 方案 1：稳定 PWM 信号
# 在软件中添加延时，确保 PWM 信号稳定

# 方案 2：改善电源
# 在 5V 电源上加 100uF 电解电容
# 使用独立的 5V 电源供电

# 方案 3：减小负载
# 检查门锁机构是否顺畅
# 减少机械阻力

# 方案 4：更换舵机
# 如果齿轮磨损，更换新舵机
```

## 5️⃣ IR 传感器问题

### 问题 5.1：IR 传感器无反应

**现象：**
- 手放在传感器前无反应
- 程序不触发人脸扫描

**可能原因：**
1. IR 传感器驱动未加载
2. 传感器损坏
3. 检测距离设置不当
4. 接线错误

**诊断步骤：**

```bash
# 1. 检查驱动加载
lsmod | grep ir_sensor
# 应该看到 ir_sensor 模块

# 2. 检查设备节点
ls -l /dev/ir_sensor
# 应该存在

# 3. 手动读取传感器
cat /dev/ir_sensor
# 手放在传感器前，应该看到 0 变为 1

# 4. 查看传感器 LED
# 红外发射管应该发出红光（肉眼不可见，用手机摄像头可以看到）
```

**解决方案：**

```bash
# 方案 1：检查接线
# 参考 docs/硬件接线图.md
# 确认 OUT 接 GPIO5_IO02

# 方案 2：调节检测距离
# 使用螺丝刀调节传感器上的电位器
# 顺时针增加距离，逆时针减少距离

# 方案 3：更换传感器测试
# 确认传感器本身无问题

# 方案 4：检查电源
# 确认 VCC 接 3.3V 或 5V
```

---

### 问题 5.2：IR 传感器误触发

**现象：**
- 无人时传感器误触发
- 频繁触发人脸扫描

**可能原因：**
1. 环境光干扰
2. 反射物体干扰
3. 检测灵敏度过高
4. 电源干扰

**诊断步骤：**

```bash
# 1. 观察环境
# 检查传感器前方是否有反射物体

# 2. 持续监测传感器
cat /dev/ir_sensor
# 观察是否有异常变化

# 3. 查看程序日志
tail -f /opt/visionpass/logs/run.log
# 查看触发记录
```

**解决方案：**

```bash
# 方案 1：降低灵敏度
# 调节传感器上的电位器
# 逆时针旋转降低灵敏度

# 方案 2：避免干扰
# 移除传感器前方的反射物体
# 避免阳光直射传感器

# 方案 3：软件去抖
# 修改 IRSensorMonitor.cpp
# 添加软件滤波，连续检测到 N 次才触发
```

## 6️⃣ 人脸识别问题

### 问题 6.1：无法检测到人脸

**现象：**
- 摄像头画面正常但无法检测人脸
- 人脸框不显示

**可能原因：**
1. 模型文件缺失
2. 光线条件不佳
3. 人脸角度过大
4. 检测阈值设置过高

**诊断步骤：**

```bash
# 1. 检查模型文件
ls -l /opt/visionpass/model/
# 应该存在：
# - haarcascade_frontalface_alt2.xml
# - MobileFaceNet.param
# - MobileFaceNet.bin

# 2. 查看程序日志
tail -f /opt/visionpass/logs/run.log | grep -i face
# 查看人脸检测相关信息

# 3. 检查光线条件
# 确认环境光线充足
```

**解决方案：**

```bash
# 方案 1：确保模型文件存在
# 重新部署模型文件
scp model/* root@192.168.1.100:/opt/visionpass/model/

# 方案 2：改善光线条件
# 增加环境照明
# 避免逆光

# 方案 3：调整检测阈值
# 修改 FaceDetector.cpp 中的阈值
# 降低阈值可以增加检测率（但可能增加误检）

# 方案 4：优化摄像头角度
# 确保摄像头正对人脸
# 避免过大角度
```

### 问题 6.2：人脸识别失败

**现象：**
- 能检测到人脸但无法识别
- 提示"陌生人"

**可能原因：**
1. 用户未注册
2. 特征提取失败
3. 匹配阈值设置过高
4. 光线变化过大

**诊断步骤：**

```bash
# 1. 检查用户数据库
sqlite3 /opt/visionpass/data/users.db "SELECT * FROM users;"
# 查看已注册的用户

# 2. 查看程序日志
tail -f /opt/visionpass/logs/run.log | grep -i recognize
# 查看识别过程和相似度分数

# 3. 检查特征提取
# 查看 NCNN 模型是否正常加载
```

**解决方案：**

```bash
# 方案 1：注册用户
# 使用用户管理界面注册人脸

# 方案 2：降低匹配阈值
# 修改 SystemController.cpp 中的阈值
# 从 0.6 降低到 0.5

# 方案 3：重新注册
# 在不同光线条件下重新注册人脸
# 注册多张照片提高识别率

# 方案 4：检查模型文件
# 确认 MobileFaceNet 模型文件完整
```

## 7️⃣ 程序崩溃问题

### 问题 7.1：程序运行一段时间后崩溃

**现象：**
- 程序运行一段时间后突然退出
- 可能伴随内存不足错误

**可能原因：**
1. 内存泄漏
2. 线程竞争
3. 文件描述符泄漏
4. 堆栈溢出

**诊断步骤：**

```bash
# 1. 监控内存使用
while true; do free -m; sleep 1; done
# 观察内存是否持续下降

# 2. 查看程序日志
tail -n 100 /opt/visionpass/logs/run.log
# 查看崩溃前的日志

# 3. 使用 gdb 调试
gdb /opt/visionpass/bin/visionpass
(gdb) run
# 等待崩溃
(gdb) bt
# 查看调用栈

# 4. 检查文件描述符
ls -l /proc/$(pgrep visionpass)/fd
# 查看打开的文件描述符数量
```

**解决方案：**

```bash
# 方案 1：修复内存泄漏
# 使用 valgrind 检测（如果开发板支持）
# 或者在代码中检查 new/delete 配对

# 方案 2：增加内存监控
# 在代码中定期检查内存使用
# 超过阈值时释放缓存

# 方案 3：优化资源管理
# 确保所有线程正确退出
# 确保所有文件描述符正确关闭

# 方案 4：使用 systemd 自动重启
# 配置 Restart=always
# 参考部署手册中的 systemd 配置
```

## 8️⃣ 性能问题

### 问题 8.1：程序运行缓慢

**现象：**
- 界面响应慢
- 识别速度慢
- CPU 使用率高

**可能原因：**
1. CPU 负载过高
2. 内存不足
3. 算法效率低
4. I/O 瓶颈

**诊断步骤：**

```bash
# 1. 检查 CPU 使用率
top
# 查看各进程 CPU 使用率

# 2. 检查内存使用
free -m
# 查看可用内存

# 3. 检查 I/O 使用
iostat -x 1
```

## 📊 常用调试命令

### 系统监控

```bash
# CPU 和内存监控
top

# 内存详情
free -m

# 磁盘使用
df -h

# 进程列表
ps aux

# 系统负载
uptime
```

### 内核日志

```bash
# 查看所有内核日志
dmesg

# 实时查看
dmesg -w

# 过滤特定关键词
dmesg | grep -E "rc522|servo|ir_sensor|uvc"
```

### 程序调试

```bash
# 查看程序输出
/opt/visionpass/bin/visionpass 2>&1 | tee output.log

# 使用 strace 跟踪系统调用
strace -f /opt/visionpass/bin/visionpass

# 使用 gdb 调试
gdb /opt/visionpass/bin/visionpass

# 查看程序依赖
ldd /opt/visionpass/bin/visionpass
```

### 数据库操作

```bash
# 查看用户表
sqlite3 /opt/visionpass/data/users.db "SELECT * FROM users;"

# 删除用户
sqlite3 /opt/visionpass/data/users.db "DELETE FROM users WHERE id='xxx';"

# 清空数据库
rm /opt/visionpass/data/users.db
```
