## 系统架构

VisionPass 采用 **PC端管理 + 开发板识别** 的分离架构：

```
┌─────────────────────┐         ┌─────────────────────┐
│   PC端（Ubuntu）    │         │  开发板（I.MX6ULL） │
│                     │         │                     │
│  user_manager       │  同步   │  visionpass         │
│  ├─ 添加用户        │ ──────> │  ├─ 人脸识别        │
│  ├─ 注册人脸        │  DB     │  ├─ RFID刷卡        │
│  ├─ 管理RFID卡      │         │  ├─ 密码验证        │
│  └─ 修改密码        │         │  ├─ 语音识别        │
│                     │         │  └─ 舵机控制        │
└─────────────────────┘         └─────────────────────┘
```

**优势**：
- PC端性能更强，人脸特征提取更快
- 开发板只负责识别，程序更轻量
- 便于批量管理用户

## 用户管理

### 添加用户

1. 切换到 **"用户管理"** 标签页
2. 点击 **"添加用户"** 按钮
3. 填写用户信息：
   - **用户ID**：唯一标识（如：`user001`）
   - **用户名**：显示名称（如：`张三`）
   - **密码**：用于密码开锁（如：`123456`）
4. 点击 **"确定"**

**示例**：
```
用户ID:   user001
用户名:   张三
密码:     123456
```

### 删除用户

1. 在用户列表中选择要删除的用户
2. 点击 **"删除用户"** 按钮
3. 确认删除

**注意**：删除用户会同时删除该用户的人脸特征和RFID卡记录。

### 查看用户列表

用户列表会自动刷新，显示：
- 用户ID
- 用户名
- 创建时间
- 操作按钮

## 人脸注册

### 照片要求

- ✅ **正面照**：脸部朝向正前方
- ✅ **光线均匀**：避免强光或阴影
- ✅ **清晰度**：分辨率建议 640x480 以上
- ✅ **人脸占比**：人脸占照片的 1/3 以上
- ❌ **避免**：侧脸、遮挡、模糊、逆光

### 注册步骤

1. 切换到 **"人脸注册"** 标签页
2. 输入要注册的 **用户ID**（必须先在"用户管理"中添加）
3. 点击 **"选择照片"** 按钮
4. 选择一张符合要求的人脸照片
5. 程序自动处理：
   - 检测人脸位置
   - 裁剪人脸区域
   - 缩放到 112x112 像素
   - 提取 128 维特征向量
6. 点击 **"保存特征"** 保存到数据库

### 多张照片注册

为提高识别准确率，可以为同一用户注册多张照片（不同角度）：

1. 注册第一张正面照
2. 选择 **"追加特征"** 而不是"覆盖"
3. 依次添加左侧、右侧、略微仰角等照片

**建议**：注册 3-5 张不同角度的照片。

## RFID卡管理

### 注册RFID卡

1. 切换到 **"RFID卡管理"** 标签页
2. 点击 **"读取新卡"** 按钮
3. 将RFID卡靠近开发板的RC522模块
4. 程序自动读取卡号（8位十六进制，如：`AABBCCDD`）
5. 输入要绑定的 **用户ID**
6. 点击 **"注册卡片"**

### 手动输入卡号

如果无法直接读取，可以手动输入：
1. 在 **卡号** 输入框中输入8位十六进制卡号
2. 输入 **用户ID**
3. 点击 **"注册卡片"**

### 删除RFID卡

1. 在卡列表中选择要删除的卡
2. 点击 **"删除卡片"** 按钮
3. 确认删除

## 密码管理

### 修改密码

1. 切换到 **"密码管理"** 标签页
2. 输入 **用户ID**
3. 输入 **新密码**
4. 点击 **"修改密码"**

**注意**：
- 密码使用 SHA-256 加密存储
- 修改后需要重新同步数据库到开发板

### 密码验证

开发板上的密码验证流程：
1. 用户输入密码
2. 程序计算 SHA-256 哈希
3. 与数据库中的哈希值比对
4. 匹配则开锁

## 同步数据库到开发板

### 方法1：SCP命令（推荐）

```bash
# 从PC端复制到开发板
scp -o StrictHostKeyChecking=no \
    -o HostKeyAlgorithms=+ssh-rsa \
    -o PubkeyAcceptedKeyTypes=+ssh-rsa \
    /home/viper/linux/visionpass/tools/user_manager/build/users.db \
    root@192.168.0.102:/opt/visionpass/data/users.db
```

### 方法2：使用脚本

创建同步脚本 `sync_db.sh`：

```bash
#!/bin/bash
PC_DB="/home/viper/linux/visionpass/tools/user_manager/build/users.db"
BOARD_DB="root@192.168.0.102:/opt/visionpass/data/users.db"

echo "正在同步数据库到开发板..."
scp -o StrictHostKeyChecking=no \
    -o HostKeyAlgorithms=+ssh-rsa \
    -o PubkeyAcceptedKeyTypes=+ssh-rsa \
    "$PC_DB" "$BOARD_DB"

if [ $? -eq 0 ]; then
    echo "✓ 数据库同步成功"
else
    echo "✗ 数据库同步失败"
fi
```

使用方法：
```bash
chmod +x sync_db.sh
./sync_db.sh
```

### 方法3：图形化工具

使用 FileZilla 或 WinSCP：
1. 连接到开发板（IP: 192.168.0.102）
2. 导航到 PC 端数据库路径
3. 拖拽到开发板的 `/opt/visionpass/data/` 目录

## 测试验证

### 1. 重启开发板程序

同步数据库后，在开发板上重启程序：

```bash
# SSH登录开发板
ssh root@192.168.0.102

# 杀掉旧进程
pkill -9 visionpass

# 等待2秒
sleep 2

# 启动程序（环境变量已自动配置）
/opt/visionpass/bin/visionpass
```

### 2. 验证人脸特征加载

观察启动日志，应该看到：
```
UserDatabase: Opened /opt/visionpass/data/users.db
UserDatabase: Ready
FaceProcessThread: Loaded 1 face features from database
```

如果显示 `Loaded 0 face features`，说明数据库同步失败或用户未注册人脸。

### 3. 测试人脸识别

1. 站在摄像头前（距离 30-50cm）
2. 正面朝向摄像头
3. 观察日志输出：

**识别成功**：
```
FaceRecognizer: Best match = user001, similarity = 0.78, threshold = 0.60
SystemController: State changed to 10  (UNLOCKED)
ServoControl: Angle set to 90
Door unlocked, auto-lock in 5000 ms
```

**识别失败**：
```
FaceRecognizer: Best match = unknown, similarity = 0.45, threshold = 0.60
FaceProcessThread: Unknown person
```

### 4. 测试RFID刷卡

1. 将已注册的RFID卡靠近RC522模块
2. 观察日志：

**刷卡成功**：
```
RFIDThread: Card detected: AABBCCDD
SystemController: State changed to 10
ServoControl: Angle set to 90
Door unlocked, auto-lock in 5000 ms
```

**未授权卡**：
```
RFIDThread: Unauthorized card: 11223344
```

### 5. 测试密码开锁

1. 点击触摸屏上的 **"密码开锁"** 按钮
2. 输入用户密码（如：`123456`）
3. 点击 **"确认"**

**密码正确**：
```
Password submitted, length= 6
Password correct, unlocking door
SystemController: State changed to 10
ServoControl: Angle set to 90
```

**密码错误**：
```
Password submitted, length= 6
Password incorrect
```

## 常见问题

### Q1: 运行 user_manager 报错 "cannot connect to X server"

**原因**：PC端没有图形环境（如SSH远程登录）

**解决**：
```bash
# 使用X11转发
ssh -X user@pc_ip
./user_manager

# 或使用VNC远程桌面
```

### Q2: 人脸注册时提示 "未检测到人脸"

**原因**：照片不符合要求

**解决**：
- 使用正面清晰照片
- 确保光线充足
- 人脸占照片 1/3 以上
- 尝试其他照片

### Q3: 开发板显示 "Loaded 0 face features"

**原因**：数据库未同步或路径错误

**检查**：
```bash
# 检查数据库文件是否存在
ls -lh /opt/visionpass/data/users.db

# 检查数据库内容
sqlite3 /opt/visionpass/data/users.db "SELECT COUNT(*) FROM face_features;"
# 应输出注册的人脸数量
```

**解决**：重新同步数据库（见[同步数据库到开发板](#同步数据库到开发板)）

### Q4: 人脸识别总是失败（similarity < 0.6）

**原因**：
1. 注册照片与实时图像差异太大
2. 光线条件不同
3. 角度差异过大

**解决**：
1. 注册多张不同角度的照片
2. 确保识别时光线与注册时相近
3. 调整识别阈值（见下文）

### Q5: 如何调整识别阈值？

**当前阈值**：0.6（相似度 > 60% 认为匹配）

**调整方法**：

编辑开发板程序源码：
```bash
# 打开SystemController.cpp
vi /home/viper/linux/visionpass/app/src/controller/SystemController.cpp
```

找到初始化代码（约第200行）：
```cpp
m_recognizer = new FaceRecognizer(this);
m_recognizer->setThreshold(0.6f);  // 修改这里的值
```

**建议值**：
- `0.5f`：宽松（容易识别，可能误识别）
- `0.6f`：平衡（默认值）
- `0.7f`：严格（减少误识别，可能漏识别）

重新编译并部署：
```bash
cd /home/viper/linux/visionpass/app/build-arm
make -j4
scp visionpass root@192.168.0.102:/opt/visionpass/bin/
```

### Q6: RFID刷卡无反应

**检查步骤**：

1. 检查设备节点：
```bash
ls -l /dev/rc522
```

2. 检查驱动加载：
```bash
lsmod | grep rc522
```

3. 查看日志：
```bash
tail -f /opt/visionpass/logs/visionpass.log | grep RFID
```

4. 手动测试：
```bash
/opt/visionpass/bin/rc522_test
# 将卡靠近模块，观察是否输出卡号
```

### Q7: 舵机不转动

**检查步骤**：

1. 检查设备节点：
```bash
ls -l /dev/servo
```

2. 手动测试：
```bash
/opt/visionpass/bin/servo_test 90
# 观察舵机是否转到90度
```

3. 检查PWM信号：
```bash
cat /sys/class/pwm/pwmchip0/pwm0/duty_cycle
# 应该看到变化的值
```

4. 检查接线：
   - 红色线 → 5V
   - 棕色线 → GND
   - 橙色线 → GPIO1_IO27（JP6 Pin 5）

### Q8: 每次SSH都要手动 export 环境变量

**原因**：环境变量未写入配置文件

**解决**：已自动配置到 `/etc/profile`，重新SSH登录即可：
```bash
# 退出当前SSH
exit

# 重新登录
ssh root@192.168.0.102

# 验证环境变量
echo $QT_QPA_PLATFORM
# 应输出: linuxfb:fb=/dev/fb0
```

## 附录：数据库结构

### users 表
```sql
CREATE TABLE users (
    id TEXT PRIMARY KEY,           -- 用户ID
    name TEXT,                     -- 用户名
    password_hash TEXT,            -- 密码SHA-256哈希
    created_at TIMESTAMP           -- 创建时间
);
```

### face_features 表
```sql
CREATE TABLE face_features (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id TEXT,                  -- 用户ID（外键）
    feature BLOB,                  -- 128维特征向量（512字节）
    created_at TIMESTAMP,
    FOREIGN KEY(user_id) REFERENCES users(id)
);
```

### rfid_cards 表
```sql
CREATE TABLE rfid_cards (
    card_id TEXT PRIMARY KEY,      -- 卡号（8位十六进制）
    user_id TEXT,                  -- 用户ID（外键）
    created_at TIMESTAMP,
    FOREIGN KEY(user_id) REFERENCES users(id)
);
```
