# VisionPass 部署手册

> 将项目部署到 I.MX6ULL 开发板的完整指南

---

## 📦 部署前准备

### 硬件清单
- ✅ I.MX6ULL 开发板（正点原子阿尔法版）
- ✅ 电源适配器（5V/2A）
- ✅ 网线或 WiFi 模块
- ✅ USB 摄像头（UVC 兼容）
- ✅ RC522 RFID 模块
- ✅ SG90 舵机
- ✅ IR 红外传感器
- ✅ 蜂鸣器（板载或外接）
- ✅ RFID 卡片（Mifare 1K）

### 软件环境
- ✅ 开发板已刷入 Yocto/Buildroot 系统
- ✅ 开发板 IP：192.168.1.100（可自定义）
- ✅ PC IP：192.168.1.101
- ✅ SSH 可连接开发板
- ✅ NFS 服务已配置（可选）

## 🔧 步骤 1：编译项目

### 1.1 在 PC 上交叉编译

```bash
# 进入项目目录
cd /home/viper/linux/visionpass/app

# 创建构建目录
mkdir -p build-arm && cd build-arm

# 配置 CMake（使用交叉编译工具链）
cmake -DCMAKE_TOOLCHAIN_FILE=../cmake/arm-linux-gnueabihf.toolchain.cmake ..

# 编译（使用 4 核并行）
make -j4

# 检查生成的可执行文件
file visionpass
# 输出应包含：ELF 32-bit LSB executable, ARM
```

### 1.2 编译内核驱动模块

```bash
# 编译 RC522 驱动
cd /home/viper/linux/visionpass/drivers/rc522
make

# 编译舵机驱动
cd /home/viper/linux/visionpass/drivers/servo
make

# 编译 IR 传感器驱动
cd /home/viper/linux/visionpass/drivers/ir_sensor
make

# 检查生成的 .ko 文件
ls -lh *.ko
```

## 📤 步骤 2：传输文件到开发板

### 方法 1：使用 SCP（推荐）

```bash
# 在开发板上创建目录
ssh root@192.168.1.100 "mkdir -p /opt/visionpass/{bin,drivers,model,data,logs}"

# 传输主程序
cd /home/viper/linux/visionpass/app/build-arm
scp visionpass root@192.168.1.100:/opt/visionpass/bin/

# 传输驱动模块
cd /home/viper/linux/visionpass/drivers
scp rc522/rc522.ko root@192.168.1.100:/opt/visionpass/drivers/
scp servo/servo.ko root@192.168.1.100:/opt/visionpass/drivers/
scp ir_sensor/ir_sensor.ko root@192.168.1.100:/opt/visionpass/drivers/

# 传输模型文件
cd /home/viper/linux/visionpass
scp model/MobileFaceNet.param root@192.168.1.100:/opt/visionpass/model/
scp model/MobileFaceNet.bin root@192.168.1.100:/opt/visionpass/model/
scp model/haarcascade_frontalface_alt2.xml root@192.168.1.100:/opt/visionpass/model/

# 设置执行权限
ssh root@192.168.1.100 "chmod +x /opt/visionpass/bin/visionpass"
```

### 方法 2：使用 NFS（开发调试用）

```bash
# PC 端：配置 NFS 共享
sudo vim /etc/exports
# 添加：/home/viper/linux/visionpass *(rw,sync,no_root_squash)

sudo systemctl restart nfs-kernel-server

# 开发板端：挂载 NFS
ssh root@192.168.1.100
mount -t nfs 192.168.1.101:/home/viper/linux/visionpass /mnt/nfs

# 创建符号链接
ln -sf /mnt/nfs/app/build-arm/visionpass /opt/visionpass/bin/visionpass
ln -sf /mnt/nfs/drivers/*.ko /opt/visionpass/drivers/
ln -sf /mnt/nfs/model/* /opt/visionpass/model/
```

## 🔌 步骤 3：硬件接线

### 3.1 RC522 RFID 模块接线

| RC522 引脚 | I.MX6ULL 引脚 | 说明 |
|-----------|--------------|------|
| VCC | 3.3V | 电源（不要接 5V！） |
| GND | GND | 地线 |
| RST | GPIO1_IO09 | 复位引脚 |
| SPI_CS | GPIO1_IO08 | 片选引脚 |
| SPI_SCK | ECSPI3_SCLK | SPI 时钟 |
| SPI_MISO | ECSPI3_MISO | SPI 主入从出 |
| SPI_MOSI | ECSPI3_MOSI | SPI 主出从入 |
| IRQ | 不接 | 中断引脚（未使用） |

### 3.2 SG90 舵机接线

| 舵机线色 | I.MX6ULL 引脚 | 说明 |
|---------|--------------|------|
| 红色 | 5V | 电源 |
| 棕色 | GND | 地线 |
| 橙色 | GPIO1_IO06 | PWM 控制线 |

### 3.3 IR 红外传感器接线

| 传感器引脚 | I.MX6ULL 引脚 | 说明 |
|-----------|--------------|------|
| VCC | 3.3V | 电源 |
| GND | GND | 地线 |
| OUT | GPIO5_IO02 | 信号输出 |

### 3.4 USB 摄像头

直接插入开发板的 USB 接口，系统会自动识别为 `/dev/video0`

## 🚀 步骤 4：加载驱动并运行

### 4.1 SSH 登录开发板

```bash
ssh root@192.168.1.100
```

### 4.2 加载内核驱动模块

```bash
# 加载 RC522 驱动
insmod /opt/visionpass/drivers/rc522.ko

# 加载舵机驱动
insmod /opt/visionpass/drivers/servo.ko

# 加载 IR 传感器驱动
insmod /opt/visionpass/drivers/ir_sensor.ko

# 验证驱动加载成功
lsmod | grep -E "rc522|servo|ir_sensor"

# 检查设备节点
ls -l /dev/rc522 /dev/servo /dev/ir_sensor
```

### 4.3 创建数据库目录

```bash
mkdir -p /opt/visionpass/data
chmod 777 /opt/visionpass/data
```

### 4.4 设置环境变量

```bash
# 设置 Qt 平台插件为 linuxfb（帧缓冲）
export QT_QPA_PLATFORM=linuxfb:fb=/dev/fb0

# 设置 Qt 插件路径
export QT_PLUGIN_PATH=/usr/lib/qt5/plugins

# 设置库路径（如果使用自定义库）
export LD_LIBRARY_PATH=/opt/visionpass/lib:$LD_LIBRARY_PATH
```

### 4.5 运行主程序

```bash
# 直接运行
/opt/visionpass/bin/visionpass

# 或者带调试输出运行
/opt/visionpass/bin/visionpass -platform linuxfb:fb=/dev/fb0 2>&1 | tee /opt/visionpass/logs/run.log

# 后台运行
/opt/visionpass/bin/visionpass &
```

## 🔄 步骤 6：配置开机自启动（可选）

### 6.1 创建 systemd 服务

```bash
# 在开发板上创建服务文件
vim /etc/systemd/system/visionpass.service
```

添加以下内容：

```ini
[Unit]
Description=VisionPass AI Access Control System
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/visionpass

# 加载驱动
ExecStartPre=/sbin/insmod /opt/visionpass/drivers/rc522.ko
ExecStartPre=/sbin/insmod /opt/visionpass/drivers/servo.ko
ExecStartPre=/sbin/insmod /opt/visionpass/drivers/ir_sensor.ko

# 设置环境变量
Environment="QT_QPA_PLATFORM=linuxfb:fb=/dev/fb0"
Environment="QT_PLUGIN_PATH=/usr/lib/qt5/plugins"

# 启动主程序
ExecStart=/opt/visionpass/bin/visionpass

# 崩溃后自动重启
Restart=always
RestartSec=5

# 停止时卸载驱动
ExecStopPost=/sbin/rmmod rc522
ExecStopPost=/sbin/rmmod servo
ExecStopPost=/sbin/rmmod ir_sensor

[Install]
WantedBy=multi-user.target
```

### 6.2 启用服务

```bash
# 重新加载 systemd 配置
systemctl daemon-reload

# 启用开机自启动
systemctl enable visionpass.service

# 立即启动服务
systemctl start visionpass.service

# 查看服务状态
systemctl status visionpass.service

# 查看日志
journalctl -u visionpass.service -f
```

## 🐛 步骤 7：常见问题排查

### 问题 1：程序启动后黑屏

**可能原因：**
- Qt 平台插件未设置
- 帧缓冲设备不存在
- 显卡驱动未加载

**解决方法：**
```bash
# 检查帧缓冲设备
ls -l /dev/fb0

# 设置正确的平台插件
export QT_QPA_PLATFORM=linuxfb:fb=/dev/fb0

# 检查显卡驱动
lsmod | grep drm
```

### 问题 2：摄像头无法打开

**可能原因：**
- 摄像头未正确连接
- UVC 驱动未加载
- 权限问题

**解决方法：**
```bash
# 检查 USB 设备
lsusb

# 检查视频设备
ls -l /dev/video*

# 检查 UVC 驱动
lsmod | grep uvcvideo

# 手动加载 UVC 驱动
modprobe uvcvideo

# 检查权限
chmod 666 /dev/video0
```

### 问题 3：RFID 读卡无反应

**可能原因：**
- 驱动未加载
- SPI 设备配置错误
- 接线错误

**解决方法：**
```bash
# 检查驱动是否加载
lsmod | grep rc522

# 检查设备节点
ls -l /dev/rc522

# 查看内核日志
dmesg | grep rc522

# 检查 SPI 设备
ls -l /dev/spidev*
```

### 问题 4：舵机不转动

**可能原因：**
- 驱动未加载
- PWM 配置错误
- 电源不足

**解决方法：**
```bash
# 检查驱动是否加载
lsmod | grep servo

# 检查设备节点
ls -l /dev/servo

# 查看内核日志
dmesg | grep servo

# 检查舵机电源（应该是 5V，不是 3.3V）
```

### 问题 5：程序崩溃

**可能原因：**
- 内存不足
- 段错误
- 依赖库缺失

**解决方法：**
```bash
# 查看程序日志
cat /opt/visionpass/logs/run.log

# 使用 gdb 调试
gdb /opt/visionpass/bin/visionpass
(gdb) run
(gdb) bt  # 查看调用栈

# 检查内存使用
free -m
top
```
